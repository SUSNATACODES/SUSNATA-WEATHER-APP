Project

A real weather product

Not just a temperature card. It has live intelligence, forecast, air quality, mail reports, contact and backup systems.

Hosting

Blogger plus Render

Blogger keeps the public site available. Render protects private API keys, mail delivery and scheduled report logic.

Backup

Still works if Render sleeps

Weather falls back to Open-Meteo and contact falls back to a Gmail or mail app draft.

Presentation

How I made it

  1. Designed the dashboard in HTML with header, radar hero, weather panels, mail alerts, contact and footer.
  2. Created the glass style, dark atmosphere, animations and mobile layout in CSS.
  3. Added JavaScript for city search, auto location, backup weather, login memory, favorites and contact fallback.
  4. Built the Express backend to protect OpenWeather keys, cache data, send mail and run alert/report checks.
  5. Connected Brevo email API, Google login config, Render deployment and Blogger XML publishing.
  6. Published the live demo, README, screenshot, presentation and source code on GitHub.

Architecture

How it works

Visitor -> Blogger frontend -> Render Express API -> OpenWeather -> Dashboard
If Render sleeps -> Blogger frontend -> Open-Meteo backup
Email -> Render -> Brevo API -> Inbox
Contact fallback -> Gmail/mail draft

Feature Set

What is included

Animated UI, auto location, OpenWeather, Open-Meteo backup, Google login, Gmail-style reminders, important alerts, 12:00 AM reports, contact fallback, Blogger XML, Render backend and GitHub documentation.
